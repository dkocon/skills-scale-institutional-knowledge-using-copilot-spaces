# Role Handoff / Handover Checklist

Purpose: Standardize cross-role handoffs to reduce ambiguity and failures at boundaries between discovery, delivery, and operations.

For each handoff, fill:
- From role:
- To role:
- Date/time:
- Context (PR/Issue/Release ID):
- Expected deliverables:

Common handoffs:
1) Discovery -> Delivery (Product Manager -> Dev Team)
   - Deliverables: One-pager, prioritized backlog items, acceptance criteria, success metrics, user research artifacts
2) Development -> QA (Dev -> QA Lead)
   - Deliverables: Test plan, feature toggles, environment instructions, known limitations
3) QA -> Release (QA -> Release Engineer)
   - Deliverables: Sign-off document, smoke test steps, rollback instructions
4) Release -> Support (Release Engineer -> Support Lead)
   - Deliverables: Post-release monitoring plan, runbooks, escalation contacts

Template:
- Checklist item (yes/no)
- Owner
- Notes / follow-up actions
- Sign-off (name/date)
